# AdaptiveGlyphKit 0.1.0 Release Candidate

Intended public SwiftPM coordinate:
`https://github.com/joshlacal/AdaptiveGlyphKit.git`, planned release `0.1.0`.
Neither the repository URL nor the immutable tag is published yet.
Published library manifests use `.upToNextMinor(from: "0.1.0")`; exact
resolution is reserved for the disposable tag-verification consumer.

AdaptiveGlyphKit 0.1.0 forges bounded source-image input on iOS 18, macOS 15,
Mac Catalyst 18, tvOS 18, and visionOS 2. watchOS 11 consumes and renders
pre-forged adaptive-glyph content; source-image forging is compile-time
unavailable there.

Pre-forged content is limited to 1,048,576 bytes, one through eight
representations, 1...1,024 pixels per edge, and 1,048,576 cumulative pixels
with checked arithmetic. Source forging defaults to 512 pixels and hard-clamps
at 1,024 pixels.

All image decode, resize, HEIC encode, structural validation, glyph parsing,
and attributed-string bridging are synchronous. Prepare and cache values
outside SwiftUI body.

The automated release matrix covers macOS and iOS tests,
Catalyst/tvOS/visionOS forge-consume runtime smoke, and watchOS consume/render
tests on Xcode 16.0 with watchOS 11.0 plus newest-stable Xcode. Those hosted
minimum and newest-stable results remain pending until the five Plan 5 workflow
markers are synchronized to the sealed runner and the jobs execute. Local Xcode
27 beta validation is advisory prerelease evidence only.

Physical-watch smoke: not performed; no physical-watch result required for
0.1.0.
