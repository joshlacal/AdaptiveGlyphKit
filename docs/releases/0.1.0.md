# AdaptiveGlyphKit 0.1.0

Public SwiftPM coordinate:
`https://github.com/joshlacal/AdaptiveGlyphKit.git`, release `0.1.0`.
Library manifests should use `.upToNextMinor(from: "0.1.0")`; exact resolution
is reserved for the disposable tag-verification consumer in CI.

AdaptiveGlyphKit 0.1.0 forges bounded source-image input on iOS 18, macOS 15,
Mac Catalyst 18, tvOS 18, and visionOS 2. watchOS 11 consumes and renders
pre-forged adaptive-glyph content; source-image forging is compile-time
unavailable there.

Pre-forged content is limited to 1,048,576 bytes, one through eight
representations, 1...1,024 pixels per edge, and 1,048,576 cumulative pixels
with checked arithmetic. Source forging defaults to 512 pixels and hard-clamps
at 1,024 pixels.

All image decode, resize, HEIC encode, structural validation, glyph parsing,
and attributed-string bridging are synchronous. Prepare and cache values
outside SwiftUI body.

Glyph construction goes through the Objective-C runtime so that content the
system rejects surfaces as `nil` instead of crashing: the imported
`NSAdaptiveImageGlyph(imageContent:)` initializer is non-failable in Swift but
returns nil at runtime, and letting that null flow into a non-optional class
value crashes under optimized builds. The `no-document-name.heic` fixture
exercises this system-rejection branch in both Debug and Release test runs.

## Verification

Continuous integration (`ci.yml`, push and pull request) runs the full suite in
Debug and Release on the minimum toolchain (Xcode 16.0) and the newest stable
Xcode on the hosted macOS 15 image, builds every supported platform in Release,
runs the platform runtime matrix (macOS and iOS full suites; Catalyst, tvOS,
and visionOS runtime smoke), runs the complete suite on the newest available
watchOS runtime, proves the watchOS compile-time API surface, and verifies
fixture provenance.

The release workflow (`release.yml`, bare-SemVer tags) additionally verifies
the exact minimum Xcode build (16A242d), downloads watchOS 11.0 and runs the
complete watch suite at the minimum deployment target, and finishes by
resolving and building the public immutable tag from a disposable clean
consumer package.

Local validation on the Xcode 27 beta toolchain is advisory prerelease
evidence only.

Physical-watch smoke: not performed; no physical-watch result is required for
0.1.0.
